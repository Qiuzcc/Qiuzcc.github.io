分支管理策略

  • 分工合作策略
master 稳定版本的分支
dev 实际工作过程中使用的主分支，dev完成阶段任务时，才把dev分支合并到master上
user1 工作分支一，在dev分支的基础上创建，完成任务后，合并到dev分支
user2 工作分支二
… …

  • 依赖分支实现的版本管理策略（串行策略）
master 最新分支
dev1 工作阶段一分支，完成后在此分支上创建dev2，合并dev1到master，切换都dev2继续工作
dev2 工作阶段二分支，同上
…
dev9 最后一个工作阶段，完成后，合并dev9到master，不必再创建新分支

这样就实现了一个分支一个版本

七、远程仓库

由于本地Git仓库和GitHub仓库之间的传输是通过SSH加密的，所以，需要设置SSH Key

创建SSH Key

在用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥

登陆GitHub，打开“Account settings”，“SSH Keys”页面

点“Add SSH Key”，填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容

点“Add Key”

查看远程仓库信息

推送分支

推送时，要指定本地分支（如：master），git会把该分支推送到远程库的对应分支上

如果是在远程仓库新建分支，可以加上-u参数，这样可以把本地分支和远程分支关联起来，下次push就可以省略远程分支名
